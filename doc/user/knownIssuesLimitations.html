The following known issues/limitations have been observed on Windows 7, Ubuntu 12.04 LTS (Precise Pangolin; both 32-bit and 64-bit) and/or OS X 10.8 (Mountain Lion). So, if you use a different operating system, these issues/limitations may or not be relevant to you.

Windows, Linux and OS X

By default, OpenCOR uses your system's language for menus, dialog boxes, etc., as long as it is either English or French (please contact us if you would like another language to be added). If your system uses another language, then OpenCOR will default to English. Otherwise, if you specifically specify English or French, then please be aware that system messages, dialog boxes, etc. will still be displayed in your system's language.

Windows
  • OpenCOR's embedded file browser may, in some cases, result in OpenCOR being slow to respond at startup. This problem has nothing to do with OpenCOR, but with Windows or a program which you use and which tightly integrates with Windows. It may not be an easy problem to fix, even though there several known causes and solutions which you might find useful.
Linux
  • Menu items that have sub-menus may not, on some versions of Gnome, have their icon displayed (should they normally have one). This has nothing to do with OpenCOR and has been discussed in places such as here and here. You can, however, change this behaviour by entering the following on the command line: 
    $ gconftool-2 --type Boolean --set /desktop/gnome/interface/menus_have_icons True
