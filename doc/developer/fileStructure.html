Source code files are, in OpenCOR, organised within a very specific file structure, as shown below. Any new (set of) file(s) is therefore expected to be added to the right folder. 

build                         // Where OpenCOR is built
cmake                         // CMake files used throughout the project
distrib                       // Files used during the packaging of OpenCOR
doc                           // User and developer documentation
 ├─ 3rdparty                  // Third-party library used for the documentation
 │   ├─ googleCodePrettify    // The Google Code Prettify library
 │   └─ jQuery                // The jQuery library
 ├─ developer                 // Developer documentation
 ├─ res                       // Resource files used by the documentation
 └─ user                      // User documentation
i18n                          // Internationalisation files
models                        // Some (real) sample models
res                           // Resource files
 ├─ boomy                     // The Boomy multiplatform icon library
 ├─ flags                     // The Printable world flags icon library
 └─ oxygen                    // The Oxygen icon library
src                           // Source code files
 ├─ 3rdparty                  // Third-party libraries statically linked to OpenCOR
 │   ├─ LibQxt                // LibQxt
 │   └─ QtSingleApplication   // QtSingleApplication
 ├─ misc                      // Files that do not fit anywhere else
 └─ plugins                   // Plugins supported by OpenCOR
     ├─ 3rdparty              // Third-party libraries as plugins
     │   ├─ LLVM              // LLVM as a plugin
     │   ├─ QJson             // QJson as a plugin
     │   ├─ QScintilla        // QScintilla as a plugin
     │   ├─ QtMmlWidget       // QtMmlWidget as a plugin
     │   ├─ Qwt               // Qwt as a plugin
     │   └─ SUNDIALS          // SUNDIALS solvers as plugins
     ├─ api                   // Plugins to use an API
     │   └─ CellMLAPI         // The CellML API as a plugin
     ├─ editing               // Plugins for the editing side of things
     ├─ misc                  // Plugins for things that do not fit anywhere else
     ├─ organisation          // Plugins for the organisation side of things
     └─ simulation            // Plugins for the simulation side of things
test                          // The test program
winConsole                    // Console version of OpenCOR (Windows only)
 ├─ build                     // Where the build files are generated
 └─ src                       // Source code files
