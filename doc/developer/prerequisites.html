The following tools are required in order to develop, build, run, test and/or package OpenCOR:

  • Git (the git package on Ubuntu): a distributed version control system. You can get a copy of OpenCOR's source code by cloning our Git repository either in read-only mode: 
    $ git clone git://github.com/opencor/opencor.git
    or in read-write mode: 
    $ git clone git@github.com:opencor/opencor.git
  • CMake 2.8.9+ (the cmake package on Ubuntu): a cross-platform build system;
  • C++ toolchain: a set of C++ tools. The toolchain is different depending on the platform you are using:
    • Windows: Visual C++ 2010 Express SP1 (for debugging, you will also need the Debugging Tools for Windows);
    • Linux: the g++ package on Ubuntu; and
    • OS X: Xcode 4.5 (you will also need to get both the command line tools and PackageMaker (from the auxiliary tools), which can be done via the Xcode | Open Developer Tool | More Developer Tools... menu in Xcode).
  • Qt 5.0: this combines Qt 5.0.0 (a cross-platform application and UI framework) and Qt Creator 2.6.1 (the supporting Qt IDE):
    • Windows (to be able to debug, you will need to configure Qt Creator via the Tools | Options... menu. From there, select Debugger | CDB, and add a symbol path by inserting a symbol server);
    • Linux 32-bit/64-bit (we would recommend sudo installing Qt, so that it gets installed under under /opt rather than under your home directory); and
    • OS X (you might want to install Qt under /Applications rather than your home folder as suggested by the installer).
  • NSIS (Windows only): a program for creating Windows installers.

Finally, you might want to update your PATH environment variable by adding the following path(s) (so that you can build OpenCOR from the command line):

  • Windows: C:\Program Files\CMake 2.8\bin (or C:\Program Files (x86)\CMake 2.8\bin if you are running a 64-bit version of Windows) and C:\Qt\Qt5.0.0\5.0.0\msvc2010\bin;
  • Linux: /opt/Qt5.0.0/5.0.0/gcc/bin (or /opt/Qt5.0.0/5.0.0/gcc_64/bin) in /etc/environment on Ubuntu; and
  • OS X: /Applications/Qt5.0.0/5.0.0/clang_64/bin in /etc/profile.
