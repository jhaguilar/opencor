Tests

As already mentioned, OpenCOR uses a plugin approach. Tests are therefore created at the plugin level, and every test must respect the following:

  • The test must be located under the test folder of a plugin (e.g. [OpenCOR]/src/plugins/misc/CellMLSupport/test);
  • The test must consist of both a .cpp and a .h file (e.g. [OpenCOR]/src/plugins/misc/CellMLSupport/test/test.cpp and [OpenCOR]/src/plugins/misc/CellMLSupport/test/test.h);
  • The test must be listed under the TESTS header in the call to the CMake's ADD_PLUGIN macro (e.g. [OpenCOR]/src/plugins/misc/CellMLSupport/CMakeLists.txt): 
    PROJECT(MyPlugin)

# Add the plugin

ADD_PLUGIN(MyPlugin
    ...
    TESTS
        test1
        test2
        test3
        ...
)
  • The test must be referenced in the main test program.
